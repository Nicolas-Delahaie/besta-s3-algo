# Projet

## Explications

L'application Besta a été conçue et imaginée à partir de zéro. C'est une application web de type évènementiel qui permet de trouver un évènement festif (fête de village notamment) à proximité.

Durant le S3 nous avons dû imaginer et développer une fonctionnalité complexe aussi bien en terme de données qu'algorithmiquement parlant.
Cette dernière, longuement cherchée et mûrement imaginée a été de pouvoir créer des cocktails d'alcool et de diluant, à l'aide des boissons renseignées par les participants d'une soirée en ayant à la fin le moins de restes possibles.

Cette fonctionnalité allie gestion des données et algorithmie, un cocktail parfait !

# Technologies

Le projet est développé en PHP et une documentation Doxygen est trouvable dans le dossier `doxygen-doc/`.

Au préalable du développement, une longue période a permi d'élaborer les algorithmes de chaque fonction utilisée dans le code. Ces algorithmes sont trouvables dans le dossier `docs/Algorithme`. Ces diagrammes UML ont été conçus grace à l'application Looping.

Pour suivre les bonnes pratiques apprises en cours, nous avons suivi une structure orientée objet (POO). Chaque objet possède ainsi sa classe.

# Lancement projet

Ce projet n'utilise pas de base de données. Ce n'est qu'un POC qui utilise des données en dur.  
Pour le lancer, il suffit d'utiliser un serveur PHP pour ouvrir `PoleDeveloppement/index.php` et le tour est joué.
